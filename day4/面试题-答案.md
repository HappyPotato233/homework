配套精简参考答案（面试背诵版）
---------------

1. 类装饰器必须实现`__init__`接收被装饰函数、`__call__`实现调用逻辑；优势是可以保存状态、自定义实例方法操作缓存 / 计数；适合缓存、接口限流、调用统计；简单无状态逻辑用普通函数装饰器。
2. time.sleep 同步阻塞整个线程，卡死事件循环，所有协程串行；asyncio.sleep 会释放事件循环，实现真正并发；异步代码必须使用 await asyncio.sleep。
3. 多线程由操作系统调度、切换开销大、内存占用高，共享数据需要加锁；协程单线程内用户态调度、开销极低，单线程无资源竞争无需加锁；少量同步 IO 用多线程，海量网络 / IO 请求用异步。
4. 不能，外部会通过 await 调用同步函数，抛出`can't await str`异常；子类重写异步抽象方法必须添加 async 关键字。
5. Thread 不会保存返回值，直接丢弃；方案：①可变容器传参接收结果 ②自定义 Thread 子类存储返回属性 ③ThreadPoolExecutor 的 Future.result ()。
6. venv 隔离项目依赖，requirements.txt 记录第三方库版本；导出`pip freeze > requirements.txt`，安装`pip install -r requirements.txt`；**init**.py 标记文件夹为 Python 包，可简化导入。
7. 字典写入非原子操作，多线程同时修改产生竞争；with lock 自动加锁释放，不用手动 acquire/release，避免异常导致死锁。
8. 网页端不能修改历史 commit；本地`git commit --amend`修改最近一次；多人分支强制推送会重写分支历史，造成其他同事本地代码冲突丢失。
9. 实例方法 self 操作实例属性；类方法 cls 操作全类共享属性；静态方法无 self/cls，纯工具计算；抽象方法仅定义接口，子类必须重写，不能实例化抽象类。
10. 不是串行，gather 一次性启动全部协程，任务完成先后随机；但返回列表严格匹配传入顺序；所有 IO 并发等待，总耗时等于耗时最长的任务。
